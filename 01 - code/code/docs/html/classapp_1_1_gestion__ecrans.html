Description détaillée

Classe qui s'occupe de gérer les écrans.

L'idée c'est que chaque écran doit d'abords être enregistré dans la fabrique à l'initialisation de l'appli L'enregistrement ne créer pas l'écran mais associe une ID et une fonction lambda dans un <map> qui le créera au moment voulu. Les actions sur les écrans ne sont enregistrés dans une liste ( m_modifEnAttente ) et réalisé qu'à la fin de la frame (appliquerModifEnAttente()). Cela permet par exemple qu'un écran se supprime lui-même sans tout faire planter.

Gère les différents écrans de l'application. C'est lui qui porte les écrans actifs du programme. qui permet de passer d'un écran à l'autre, etc.
